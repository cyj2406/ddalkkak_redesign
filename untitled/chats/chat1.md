- **전송 버튼**: 입력값 있을 때만 활성화

**3. 몽환적 오라**
- 메인 영역에 `absolute inset-0`로 깔리는 3겹 블러 그라데이션(인디고/라벤더/바이올렛), `vw` 단위 + `max-w/max-h`로 한도 설정, `translate(-50%,-50%)` 기반 keyframes로 미세 스케일 펄스. 화면 크기와 무관하게 중앙 고정.

- 간격은 `gap-3` `gap-6` `gap-10` `px-6` `py-10` 등으로, 너비는 `max-w-3xl` `max-w-5xl` `max-w-6xl`로 잡고, 타이포는 `text-[clamp(2rem,4.6vw,3rem)]` 등 뷰포트 반응형.

1. 이원화된 이미지 수집 로직 (Dual-Path Ingestion)

베이스 이미지 (Base): 채팅창에서 AI가 생성한 이미지 결과물을 클릭하거나, 이미지 하단의 [베이스로 사용] 버튼을 누르면 즉시 프롬프트 바 상단의 'B' 슬롯에 등록되게 해줘. (베이스 이미지는 단 1개만 존재 가능하며, 새로 등록 시 기존 이미지는 교체됨)

참조 이미지 (Reference): 프롬프트 바 상단의 [+] 모양의 참조 이미지 업로드 박스를 클릭하여 내 컴퓨터에서 직접 업로드해. 최대 3개까지 등록 가능하며 'R' 슬롯에 순차적으로 쌓이게 해줘.

2. 프롬프트 바 상단: 자산 관리 영역 (Asset Slot UI)

Base Slot (좌측 고정):

오렌지색 점선 테두리(border-dashed border-orange-400)로 고정된 하나의 슬롯이야.

이미지가 없을 때는 'B' 배지와 함께 "베이스 이미지"라는 연한 텍스트가 표시돼.

이미지가 등록되면 썸네일과 함께 좌측 상단에 주황색 'B' 라벨이 붙어.

Reference Slots (우측 나열):

파란색 테두리(border-blue-200)를 가진 슬롯들이야.

이미지가 등록되면 썸네일과 함께 좌측 상단에 파란색 'R' 라벨이 붙어.

공통 인터랙션: 모든 칩에는 마우스 오버 시 삭제용 'X' 버튼이 나타나고, 삭제 시 해당 슬롯은 다시 빈 상태로 돌아가야 해.

3. 프롬프트 바 및 인터랙션 (Prompt Input)

입력창 연동: 이미지 자산(B/R)이 하나라도 등록되면, 프롬프트 바는 자동으로 "선택된 이미지를 바탕으로 수정을 진행합니다"라는 상태를 인지해야 해.

디자인: 레퍼런스처럼 아주 정갈한 화이트/그레이 톤(60-30-10 규칙)을 유지하고, 포커스 시에만 메인 블루 컬러가 은은하게 돌도록 해줘.

4. 채팅창 내 이미지 액션 (Chat UI)

AI가 생성한 각 이미지 말풍선 하단에 아주 작은 툴바를 만들어줘.

[B 베이스로 지정], [다운로드] 버튼을 선형 아이콘으로 배치해, 사용자가 마음에 드는 결과물을 즉시 다음 작업의 '베이스'로 삼을 수 있게 해줘.

이 수정을 통해 "채팅 -> 생성 -> 결과물을 베이스로 지정 -> 참조 이미지 추가 -> 재채팅"으로 이어지는 끊김 없는(Seamless) 전문가용 생성 워크플로우를 완성해줘.
